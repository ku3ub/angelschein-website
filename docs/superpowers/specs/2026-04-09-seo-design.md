# SEO Design — angel-schein.online

**Date:** 2026-04-09
**Scope:** German SEO for the Angelschein Master landing page and blog
**Out of scope:** English/expat SEO (deferred), ASO (deferred), social media automation (separate spec)

---

## Goals

- Increase organic Google search visibility for German fishing license exam searches
- Rank for both broad terms ("Angelschein", "Fischerprüfung") and long-tail terms ("Angelschein Bayern lernen", "Fischerprüfung Fragen")
- Build a content channel that grows passively via automated weekly blog posts
- Zero ongoing cost and minimal manual effort after initial setup

---

## Phase 1: Landing Page Improvements (`index.html`)

### 1.1 Structured Data (JSON-LD)

Add two JSON-LD blocks inside `<head>`:

**`SoftwareApplication` schema:**
Tells Google this is an iOS app. Enables rich search results showing app name, rating, price, and category directly in the SERP.

Fields: `name`, `operatingSystem: "iOS"`, `applicationCategory: "EducationApplication"`, `offers.price: "0"`, `offers.priceCurrency: "EUR"`, `aggregateRating` (once reviews exist).

**`FAQPage` schema:**
Mirrors the visible FAQ section below. Enables expandable FAQ snippets directly in Google search results, increasing click-through rate without requiring a higher ranking position.

### 1.2 FAQ Section (visible on page)

A new section added after the existing features section. 8 general questions — no Bundesland-specific content (state-specific content belongs in blog posts for better focus).

Questions:
1. Was ist der Angelschein?
2. Wie schwer ist die Fischerprüfung?
3. Wie lange lernt man für den Angelschein?
4. Wie viele Fragen kommen in der Prüfung?
5. Was kostet der Angelschein in Deutschland?
6. Welche Themen kommen in der Prüfung vor?
7. Kann ich die Prüfung auf Englisch ablegen?
8. Wie oft kann man die Prüfung wiederholen?

Each answer: 2–3 sentences, naturally incorporating target keywords. Matches the `FAQPage` JSON-LD exactly.

### 1.3 Meta Description Tweak

Current description mentions "Bayern, Baden-Württemberg, Hamburg & Thüringen" — keep this. Add one sentence at the end referencing the exam preparation angle more directly to improve click-through rate from search results.

## Phase 2: Jekyll Blog

### 2.1 Jekyll Setup on GitHub Pages

Convert the existing repo to Jekyll while keeping `index.html` untouched. GitHub Pages has native Jekyll support — no new hosting or tooling required.

Files to add:
- `_config.yml` — site title, description, base URL, permalink structure
- `_layouts/post.html` — blog post template reusing existing CSS (same nav, colors, fonts as landing page)
- `_layouts/default.html` — shared base layout wrapping all pages
- `blog/index.html` — blog listing page at `angel-schein.online/blog`
- `_posts/` — folder where automated posts are committed

Permalink structure: `/blog/:year/:month/:slug/` — descriptive URLs that include keywords.

Jekyll auto-generates a sitemap at `/sitemap.xml` via the `jekyll-sitemap` plugin (built into GitHub Pages). Submit to Google Search Console after setup.

### 2.2 Content Strategy

Two post types, both generated by the Claude+Cron pipeline:

**Type A — State-specific guides (highest SEO value):**
One post per supported Bundesland, targeting searches that the landing page intentionally avoids.

Examples:
- "Angelschein Bayern: Alles was du wissen musst"
- "Fischerprüfung Hamburg — Ablauf, Kosten und Tipps"
- "Angelschein Baden-Württemberg bestehen"
- "Fischerprüfung Thüringen: So bereitest du dich vor"

These are priority posts — publish in the first month.

**Type B — Topic posts (long-tail traffic):**
Targeting people in the research phase, before they are ready to download an app.

Examples:
- "Fischkunde für die Angelprüfung: Die wichtigsten Arten"
- "Die häufigsten Fehler bei der Fischerprüfung"
- "Angelschein mit Kind — ab welchem Alter?"
- "Gewässerkunde: Was musst du für den Angelschein wissen?"

Publish weekly via automation after the initial state-specific posts are live.

### 2.3 Post Structure

Each post includes:
- Frontmatter: `title`, `date`, `description` (used as meta description), `image` (Unsplash URL)
- Hero image from Unsplash (see Section 2.4)
- H1 matching the title
- 3–5 H2 sections with keyword-rich subheadings
- Natural keyword density — no stuffing
- CTA at the bottom linking to App Store / landing page download button
- Word count target: 600–900 words (enough for ranking, short enough for automation quality)

## Phase 3: Claude+Cron Automation

### 3.1 Architecture

A weekly cron job (Node.js or Python script) that:

1. **Selects a topic** — from a predefined list of post ideas stored in a JSON file (`_data/post-queue.json`). Once a topic is used, it is marked as published.
2. **Fetches a hero image** — queries the Unsplash API with a keyword derived from the post topic (e.g. "fishing germany", "angler lake", "fish water"). Takes the first result's URL.
3. **Calls Claude API** — sends the topic, target keyword, Unsplash image URL, and a prompt instructing Claude to write an SEO-optimized German blog post in Markdown with proper frontmatter.
4. **Commits the file** — writes the Markdown file to `_posts/YYYY-MM-DD-slug.md` and commits to the `main` branch via the GitHub API.
5. **GitHub Pages builds** — detects the new commit and publishes the post automatically within ~30 seconds.

### 3.2 Claude API Prompt Structure

The prompt includes:
- Post topic and primary keyword
- Secondary keywords to include naturally
- Target word count (600–900 words)
- Instruction to end with a CTA linking to the app
- Instruction to include the Unsplash image as the hero
- Instruction to write in natural, friendly German (not formal)
- Example post structure (H1, H2s, CTA)

### 3.3 Post Queue

A `_data/post-queue.json` file in the repo tracks:
- `pending` topics (with suggested keyword and Unsplash search term)
- `published` topics (with date and slug)

The automation script reads this file, picks the first pending topic, and updates the file after publishing.

### 3.4 Unsplash Integration

- Use the Unsplash Source API (no API key required for simple usage) or the official API for more control
- Search terms map to post topics: state-specific posts use "fishing [state]", topic posts use relevant nature/fishing terms
- Images are hotlinked (no download/storage needed), reducing repo size

## Technical Constraints

- Hosting: GitHub Pages (free, existing)
- Domain: angel-schein.online (existing, no changes)
- Jekyll version: GitHub Pages default (no custom plugins beyond `jekyll-sitemap`)
- Automation runtime: Node.js or Python, runs on any machine or free CI (GitHub Actions)
- Claude API: `claude-haiku-4-5` recommended for cost efficiency on weekly blog generation
- Unsplash: free tier (50 requests/hour — more than sufficient)
